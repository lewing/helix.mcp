## Key Architectural Finding

**Principle correction:** Round 3 learning stated "normalize at EVERY layer," but that leads to algorithm duplication and drift.

**Correct principle:** 
- **Validate at user/input boundaries** (CLI/MCP) — for early, useful error messages
- **Canonicalize at semantic boundaries** (cache key, URL construction) — where the value's meaning is consumed
- **Centralize the canonicalization algorithm** — multiple layers invoke the shared helper; none reimplement independently

**Implication:** Extract normalization as a shared `NormalizeQueryOrder()` helper. HTTP client and cache both call it; validation layer calls it. Single source of truth prevents drift.
